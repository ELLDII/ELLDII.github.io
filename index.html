1. Разработка backend-модуля образовательной системы Э-күндөлүк

.NET 8 | PostgreSQL | EF Core | Angular | REST API | Оптимизация БД

Суть проекта: Разработка и развитие комплексного веб-модуля для профессиональных лицеев — от управления учебным процессом и автоматизации расписания до системы выставления оценок и интеграции с государственными сервисами.

Архитектура и бизнес-логика:

  • Проектирование логики: Реализовал структуру образовательных программ, учебных групп и распределения нагрузки преподавателей с привязкой к предметам и расписанию.
  • Алгоритмы учебного процесса: Разработал систему понедельного планирования и генерации расписания с проверкой конфликтов (замены преподавателей) и защитой от случайной перезаписи.
  • Матрица оценивания: Внедрил единую модель хранения всех типов оценок (семестровые, годовые, гос. экзамены) с жестким разграничением прав доступа (учитель / завуч / директор) и учетом временных окон.
  • Оптимизация производительности: Устранил проблемы N+1. Внедрил пакетную загрузку, AsNoTracking(), in-memory проекции (Snapshot-DTO) и работу с Read-Replica для моментальной генерации тяжелых отчетов.
  • API Интеграции: Настроил поэтапную, многопоточную выгрузку данных в государственный API (attestat.gov.kg) с использованием Parallel.ForEachAsync.

Примененные навыки:

Проектирование сложных реляционных структур, оптимизация ORM и SQL-запросов под высокие нагрузки, разработка отказоустойчивых интеграций с внешними REST API, профилирование бизнес-логики.

ASP.NET Core | PostgreSQL | REST API | Nginx | Logging & Monitoring

Суть участия: Обеспечение отказоустойчивой работы серверной части и реализация модулей интеграции для государственной системы финансового мониторинга.

Технические задачи:

  • Серверное администрирование: Настроил окружение веб-сервера (Nginx/Kestrel), включая конфигурацию SSL-сертификатов, управление правами доступа и лимитами запросов.

  • Логирование и мониторинг: Внедрил систему сбора и анализа системных логов. Использовал инструменты диагностики для выявления узких мест и оперативного устранения инцидентов.

3. Организация low-latency сети для управления платформой Unitree Go2

TCP/IP | UDP | DHCP | LAN/WLAN | Диагностика сетей | RJ45

Суть проекта: Организация локальной сети и обеспечение отказоустойчивой, бесперебойной передачи пакетов данных между управляющим сервером (ПК) и клиентом (роботом Unitree Go2) для выполнения сложной хореографии в реальном времени.

Этапы работ и Траблшутинг:

  • Базовая настройка: Развернул начальную сетевую инфраструктуру через Wi-Fi роутер (настройка DHCP, таблиц маршрутизации).
  • Анализ сети: Провел диагностику пропускной способности. Выявил, что для передачи потоковых данных стабильности Wi-Fi недостаточно из-за забитого радиоэфира — фиксировалась потеря пакетов и высокий пинг (jitter).
  • Реинжиниринг: Спроектировал новую топологию и перевел систему на прямое проводное соединение (кабель UTP, коннекторы RJ45), исключив беспроводной сегмент из критической цепи.
  • Оптимизация: Настроил передачу данных с учетом особенностей TCP/UDP, добившись минимальной задержки (low-latency) и максимальной скорости отклика оборудования.

Примененные навыки:

Построение LAN, глубокое понимание пропускной способности каналов связи, траблшутинг сетевых задержек, работа на физическом уровне (Layer 1).

4. Разработка панели администрирования (Конфигуратора) и радиомостов для БПЛА

Linux | RTOS | Radio Links | GUI/CLI | Утилиты администрирования

Суть проекта: Настройка отказоустойчивых беспроводных мостов передачи данных и разработка собственной утилиты (панели управления) для глубокого администрирования и удаленного мониторинга оборудования.

Этапы работ и интеграция:

  • Удаленный мониторинг (Телеметрия): Настраивал и калибровал радиоканалы для непрерывной передачи данных. Обеспечил стабильную связь для получения метрик состояния системы в реальном времени.
  • Системное программирование: Написал микропрограммное обеспечение (прошивку) для контроллера управления, работая с операционными системами реального времени (RTOS) и архитектурой Linux на низком уровне.
  • Разработка инструмента администрирования: С нуля спроектировал и создал конфигуратор с графическим (GUI) и командным (CLI) интерфейсом. Инструмент позволяет удаленно менять параметры системы, проводить диагностику и управлять оборудованием.

Примененные навыки:

Мониторинг распределенных систем, написание скриптов и полноценных утилит для автоматизации настройки, работа с удаленным доступом, профилирование параметров.

5. Интеграция IoT-устройства (ESP32) с веб-сервисами и AI через API

HTTP/HTTPS | REST API | JSON | Клиент-сервер | DNS | SSL/TLS

Суть проекта: Интеграция аппаратного IoT-устройства (на базе микроконтроллера) с современными веб-сервисами и облачными нейросетями посредством API для обеспечения интеллектуального взаимодействия с пользователем.

Этапы работ и архитектура:

  • Сетевой стек: Организовал работу сетевого уровня на микроконтроллере, включая настройку Wi-Fi клиента и работу с DNS для резолвинга доменных имен облачных сервисов.
  • Безопасность (Security): Реализовал защищенное подключение к внешним веб-серверам, внедрив поддержку HTTPS и валидацию SSL-сертификатов на стороне клиента.
  • Интеграция по API: Настроил формирование, отправку и прием REST API запросов (методы GET и POST) к серверам облачных LLM-моделей (нейросетей).
  • Обработка данных: Разработал логику десериализации и парсинга JSON-ответов от сервера для извлечения полезной нагрузки и мгновенного отклика устройства.

Примененные навыки:

Проектирование клиент-серверной архитектуры, работа со сторонними API, глубокое понимание веб-протоколов (HTTP/HTTPS) и структуры форматов обмена данными (JSON), настройка безопасных соединений (SSL/TLS).
